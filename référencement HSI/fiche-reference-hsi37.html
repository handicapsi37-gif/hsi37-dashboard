1 LES 3 OUTILS — NE PAS LES CONFONDRE

🖥️ Terminal
La fenêtre noire avec % à la fin. C'est lui qui tourne vraiment sur ton Mac. Tu y tapes les commandes Git et tu y lances le serveur.
→ Ouvrir avec Spotlight : ⌘ + Espace → "Terminal"
💬 Claude Code
La fenêtre où tu discutes en français pour faire écrire du code. Tu la lances en tapant claude dans le Terminal.
🚫 Jamais de commandes Git ici. Jamais de serveur ici.
🌐 Navigateur
Safari ou Chrome. Sert uniquement à voir le dashboard. Adresse : localhost:8002/index.html
→ Forcer le rechargement : ⌘ + Maj + R

💡 Astuce : garde deux fenêtres Terminal ouvertes — une pour le serveur (python3), une pour Git / Claude Code. Menu Terminal → Shell → Nouvelle fenêtre, ou ⌘ + N.

2 DÉMARRER UNE SESSION DE TRAVAIL

1
Aller dans le dossier du projet
cd ~/hsi37-dashboard
cd = "change directory" (changer de dossier). ~ = ton dossier personnel.
2
Vérifier qu'on est au bon endroit
ls
Tu dois voir : index.html, app.js, styles.css, config.js
3
Lancer le serveur local (pour voir le site)
python3 -m http.server 8002
⚠️ Laisser cette fenêtre Terminal ouverte — le serveur tourne tant qu'elle est ouverte. Si message "Address already in use" (port occupé) → essaie 8003, 8004…
4
Ouvrir le dashboard dans le navigateur
localhost:8002/index.html
Colle cette adresse dans la barre du navigateur (pas dans Google).
5
Lancer Claude Code (dans une 2ème fenêtre Terminal)
cd ~/hsi37-dashboard
claude
Si demandé "trust this folder" → taper 1 (oui). Pour quitter : /exit ou Ctrl + C deux fois.

3 SAUVEGARDER ET ENVOYER EN LIGNE — LA SÉQUENCE GIT

À faire après chaque étape qui fonctionne. Toujours dans le dossier du projet.

# Étape 0 — Toujours commencer ici
cd ~/hsi37-dashboard

# Étape 1 — Voir ce qui a changé
git status
⚠ VÉRIFIER : config.js NE DOIT PAS apparaître dans la liste

# Étape 2 — Préparer les fichiers
git add .
# Le point (.) = tous les fichiers modifiés, sauf ceux ignorés comme config.js

# Étape 3 — Créer la sauvegarde avec un message clair
git commit -m "Description de ce que tu as fait"
# Pas d'accents dans le message. Ex : git commit -m "Ajout membre Dupont"

# Étape 4 — Envoyer en ligne
git push origin main
# Les changements sont visibles en ligne en moins d'une minute

Voir les 5 dernières sauvegardes

git log --oneline -5

⏪ Annuler des modifications non encore sauvegardées

git restore .

⚠️ Attention : efface tout ce qui n'a pas encore été "commité" (sauvegardé). À utiliser seulement si tu as cassé quelque chose. En cas de doute → ne rien faire et demander.

4 RÉSOLUTION DE BLOCAGES

🔴 Le Terminal affiche quote> ou dquote> en boucle
Tu as collé du texte avec un guillemet mal fermé. → Appuie sur Ctrl + C pour revenir à l'invite normale.
🔴 "command not found" (commande introuvable)
Commande mal tapée ou outil absent. Vérifie l'orthographe exacte.
🔴 "Address already in use" (port déjà utilisé)
Le port 8002 est occupé. Essaie : python3 -m http.server 8003 puis va sur localhost:8003/index.html
🔴 La page ne s'affiche pas dans le navigateur
Le serveur python3 n'est pas lancé. Relancer l'étape 3 de la routine de démarrage.
🔴 Une commande tourne sans fin
Appuie sur Ctrl + C pour l'arrêter.
🔴 Terminal complètement bloqué
Ferme la fenêtre et relance une nouvelle fenêtre Terminal.

7 SUPABASE — LA BASE DE DONNÉES

Supabase = l'endroit où sont stockés les adhérents du dashboard. Ce n'est pas sur ton Mac, c'est en ligne.

QuoiDétail
Accèssupabase.com → Se connecter → Projet hsi37-dashboard
RégionParis
Clés secrètesDans config.js sur ton Mac — NE JAMAIS copier-coller en chat
Table des adhérentsMenu gauche → Table Editor → table members
Voir les connexionsMenu gauche → Authentication → Users

💡 Supabase est indépendant de Git. Modifier un adhérent dans Supabase ne nécessite pas de git push. Git sert pour le code (app.js, index.html…), pas pour les données.

Modifier un mot de passe utilisateur

1
Aller sur supabase.com → ton projet → menu gauche Authentication → Users
2
Trouver l'utilisateur → cliquer sur les 3 points à droite de la ligne
3
Cliquer Send password recovery → un email de réinitialisation part automatiquement à l'utilisateur

OU pour forcer toi-même un nouveau mot de passe : cliquer sur l'utilisateur → Update password → saisir → Save.

10 VIDER LE CACHE DU NAVIGATEUR

Le cache (= ancienne version de la page gardée en mémoire par le navigateur) peut faire croire qu'une modification n'a pas fonctionné. À faire systématiquement après chaque modification.

ActionRaccourci
Recharger sans cache (le plus utile)⌘ + Maj + R
Recharger normalement⌘ + R
Vider tout le cache (Chrome)⌘ + Maj + Suppr → cocher "Images et fichiers en cache" → Effacer

💡 Règle simple : après chaque git push, attendre 1 minute puis faire ⌘ + Maj + R sur la page pour voir les changements.

Uploader des photos

Tuile « Dons de matériel » → tableau avec barre de recherche et filtre par statut.

Modifier un don
Bouton ✏️ dans la colonne Actions → redirige vers le formulaire pré-rempli. Modifier les champs, puis « Enregistrer ».
6
Supprimer un don
Bouton 🗑️ dans la colonne Actions → modale de confirmation → « Supprimer ». Supprime le don et toutes les photos associées dans Supabase Storage. Action irréversible.
